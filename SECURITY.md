# Security Policy

We take security seriously. If you discover a potential vulnerability, please report it to us privately so we can investigate and address it promptly.

## Reporting a vulnerability

Please do not create a public issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository:

1. Go to the repository Security tab.
2. Select Report a vulnerability.
3. Submit details privately to maintainers.

For guidance on what to include, follow GitHub's recommendations:

- clear description of the issue,
- impact assessment,
- reproduction details or proof of concept,
- suggested mitigation if known.

Reference: [Privately reporting a security vulnerability](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/report-a-vulnerability/privately-reporting-a-security-vulnerability)

We will acknowledge receipt as soon as possible and coordinate remediation.
