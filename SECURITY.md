# Security Policy

We take security seriously. If you discover a potential vulnerability, report it privately so maintainers can investigate and address it promptly.

## Reporting a vulnerability

Do not create a public issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository:

1. Go to the repository Security tab.
2. Select Report a vulnerability.
3. Submit the details privately to the maintainers.

Include the following where possible:

- A clear description of the issue.
- The likely impact.
- Reproduction details or a proof of concept.
- A suggested mitigation if you already have one.

Reference: [Privately reporting a security vulnerability](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/report-a-vulnerability/privately-reporting-a-security-vulnerability)

Maintainers will acknowledge receipt as soon as practical and coordinate remediation from there.
